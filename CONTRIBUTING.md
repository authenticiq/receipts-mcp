# Contributing

Thanks for contributing.

## Before You Start

- Check for an existing issue or open one before large changes.
- Keep changes focused. Avoid unrelated refactors.
- Add or update tests and docs for any behavior change.

## Developer Certificate of Origin

This project uses the DCO instead of a CLA. Every commit must be signed off.

Use:

```bash
git commit -s
```

That adds a line like:

```text
Signed-off-by: Your Name <you@example.com>
```

## Pull Requests

- Explain the change clearly
- Link the relevant issue when applicable
- Include test coverage or a rationale if tests are not possible
- Keep CI green before requesting review

## Security Issues

Do not file public issues for vulnerabilities. Use `hello@authenticiq.ai` instead. See `SECURITY.md`.
